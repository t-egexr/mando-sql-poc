🎯 ¿Qué es esto?

Un día estaba cansado de anotar a mano cuándo tomaba mis medicamentos, y me dije:

"¿No debería poder decirle a SQL algo tan simple como: inserta estos datos desde tal día hasta tal otro?"

La respuesta fue:

ERROR: syntax not supported

Entonces lo hice igual.

💡 ¿Y qué hice?

    INSERT INTO dosage (medication_id, amount, date)
    VALUES (5, 3, FROM '2025-04-01' TO '2025-04-05');

se convierte en:

    INSERT INTO dosage (medication_id, amount, date) VALUES
    (5, 3, '2025-04-01'),
    ...
    (5, 3, '2025-04-05');

🛠️ ¿Cómo lo armé?

  • Python + Regex + MySQL Connector
  • ChatGPT como copiloto de código
  • Todo el código está en GitHub
Ver repo

🔍 ¿Por qué importa?

Centros de salud, IoT, educación, etc., insertan datos diarios todo el tiempo.

Hacerlo a mano o con scripts externos es un castigo.

¿Y si el lenguaje fuera más humano? ¿Más natural para quienes no son devs expertos?

✉️ ¿Y qué pasó?

Le mandé la idea al foro pgsql-hackers@postgresql.org, donde escriben quienes construyen el motor de PostgreSQL. Nunca pensé que responderían…

Pero me contestó Álvaro Herrera, del equipo core (¡y chileno, como yo!).

Su respuesta fue algo como:

“Eso ya se puede hacer con generate_series()…”

Y mi respuesta fue:

No es solo código.
Es sintaxis con empatía.

Fue un momento especial: una idea pequeña llegó hasta quienes construyen el corazón de una base de datos mundial.

🧰 ¿Y si quiero usarlo desde la terminal?

Puedes ejecutar una query como esta:

    INSERT INTO dosage (...) 
    VALUES (5, 3, FROM '2025-04-01' TO '2025-04-05');

Usando el CLI:

    python mando_insert_expander.py query.sql --execute

Soporta variables de entorno, errores amigables, y expansión sin ejecutar.
